## Risk Factors – Project Matabayan

### 1. Limited Cross-Browser Compatibility Testing
While automated testing is implemented, it is currently limited to the Chrome browser environment. As a result, the system may exhibit inconsistencies in functionality, layout, or performance when accessed through other browsers such as Firefox, Edge, or Safari. This limitation increases the risk of undetected compatibility issues, potentially leading to degraded user experience or inaccessible features for some users.

### 2. Server Overload and Performance Degradation
High volumes of concurrent users or inefficient backend processes may strain server resources, resulting in slow response times, request timeouts, or potential system downtime. This risk becomes more critical during peak usage periods.

### 3. Database Capacity Limitations
As data volume grows, the database may experience storage constraints or performance bottlenecks. Without proper scaling or optimization, this could lead to slower query execution, delays in data retrieval, or increased risk of system instability.

### 4. Security Vulnerabilities
The system may be exposed to various security threats such as unauthorized access, data breaches, and injection attacks. Weak authentication mechanisms, insufficient input validation, or lack of encryption can compromise both user data and overall system integrity.

### 5. Public API Unavailability or Instability
Reliance on third-party public APIs introduces risks such as service outages, rate limiting, or unexpected structural changes. These issues may disrupt system functionalities that depend on external data sources, potentially affecting overall reliability.
